# 🔮 나의 전생은?

이름을 입력하면 AI가 당신의 전생을 알려주는 흥미유발형 웹 서비스입니다.

기원전 15만년 전부터 1980년까지, 인간·동물·무생물·미생물·외계인까지 — 당신은 전생에 무엇이었을까요?

---

## 📸 미리보기

1. 이름을 입력합니다
2. 전생의 시기, 직업/존재, 사망 원인이 랜덤으로 배정됩니다
3. OpenAI가 이 정보를 바탕으로 드라마틱한 전생 스토리를 생성합니다
4. 결과를 카카오톡, 트위터, 클립보드로 공유할 수 있습니다

## 🚀 설치 및 실행

### 1. 클론

```bash
git clone git@github.com:bcjeong13/deploy-test1.git
cd deploy-test1
```

### 2. 패키지 설치

```bash
npm install
```

### 3. 환경변수 설정

프로젝트 루트에 `.env` 파일을 생성하고 OpenAI API 키를 입력합니다:

```
OPENAI_API_KEY=sk-your-api-key-here
```

> ⚠️ `.env` 파일은 `.gitignore`에 포함되어 있어 GitHub에 업로드되지 않습니다.

### 4. 서버 실행

```bash
npm start
```

브라우저에서 `http://localhost:3000` 접속

## 📦 데이터 구성

### occupations.json (직업/존재 — 300개)

각 항목은 `id`, `name`, `era` 필드로 구성됩니다.

```json
{"id": 1, "name": "피라미드 꼭대기 돌 나르는 인부", "era": "고대"}
```

**카테고리별 예시:**

| 카테고리 | 예시 |
|----------|------|
| 🧑 인간 직업 | 조선시대 어의, 로마 검투사, 1970년대 디스코 DJ |
| 🐾 동물 | 왕실 고양이, 콜로세움의 사자, 도도새 |
| 🪨 무생물 | 피사의 사탑 기울어진 쪽 벽돌, 세종대왕의 붓 |
| 🦠 미생물 | 김치 속 유산균, 흑사병을 퍼뜨린 벼룩 |
| 👽 외계인 | 안드로메다 은하 택배기사, 블랙홀 근처 등대지기 |
| 🐉 신화적 존재 | 구미호, 저승사자의 인턴, 스핑크스의 수수께끼 출제위원 |

**시대(era) 분류:**

| era 값 | 연도 범위 |
|---------|-----------|
| 선사 | 기원전 150,000 ~ 기원전 3,000 |
| 고대 | 기원전 3,000 ~ 476 |
| 삼국시대(기원전57~935) | 기원전 57 ~ 935 |
| 고려(918~1392) | 918 ~ 1392 |
| 조선(1392~1897) | 1392 ~ 1897 |
| 중세 | 476 ~ 1453 |
| 근세 | 1453 ~ 1789 |
| 근대 | 1789 ~ 1914 |
| 현대 | 1914 ~ 1980 |
| any | 기원전 150,000 ~ 1980 (제한 없음) |

### deaths.json (사망 원인 — 130개+)

유머러스한 사망 원인 문자열 배열입니다.

```json
[
  "너무 웃기다가 숨이 멈춤",
  "바나나 껍질에 미끄러져서 낭떠러지로",
  "100단계 매운맛 라면 먹고 승천",
  "자기가 만든 함정에 자기가 빠짐"
]
```

## 🔌 API 명세

### POST `/api/past-life`

전생 결과를 생성합니다.

**Request:**
```json
{
  "name": "홍길동"
}
```

**Response:**
```json
{
  "name": "홍길동",
  "year": "기원전 2450년",
  "occupation": "피라미드 꼭대기 돌 나르는 인부",
  "death": "너무 웃기다가 숨이 멈춤",
  "story": "기원전 2450년, 나일강 유역에 홍길동이라 불리는 인부가 있었으니..."
}
```

## 💡 응용 예시

### 1. 파티/모임 아이스브레이커
- 참석자들이 각자 이름을 입력하고 전생 결과를 발표
- "전생에 김치 속 유산균이었던 사람이 누구?" 같은 대화 유도

### 2. SNS 바이럴 콘텐츠
- 공유 버튼으로 트위터/카카오톡에 결과를 공유
- "나의 전생은 조선시대 도깨비! 사인: 이불킥하다 침대에서 떨어져 사망" 같은 콘텐츠 생성

### 3. 커스터마이징
- `occupations.json`에 직업 추가/수정으로 테마 변경 가능
  - 예: 게임 캐릭터 전생, 회사 직급 전생 등
- `deaths.json`에 사인 추가로 유머 코드 조절
- `server.js`의 프롬프트를 수정해서 스토리텔링 톤 변경 가능

### 4. 이벤트/프로모션 활용
- 특정 결과가 나오면 할인 쿠폰 제공
- 시즌별 테마 (할로윈: 공포 전생, 크리스마스: 산타 마을 전생)

### 5. 교육 활용
- 역사 수업에서 시대별 직업 알아보기
- 학생들이 직접 데이터를 추가하며 프로그래밍 학습
